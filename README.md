# ラクロー打刻ボタン (Chrome拡張機能)

ラクロー (https://www.raku-ro.com/) には打刻ボタンが無く、自己申告時間を手入力する必要があります。
この拡張機能はラクローの画面に**フローティング打刻ボタン**を追加し、ワンクリックで自己申告時間を記録できるようにします。

## 仕様

- 対象: `https://www.raku-ro.com/*` (ラクローのドメイン全体)
- ボタン位置: 画面右下のフローティングボタン
- 判定ロジック (ご要望通り):
  - **1日のうち1回目** の打刻 → **出勤**
  - **2回目以降** の打刻 → **退勤** (最後の打刻が退勤時刻として上書きされる想定)
- 時刻: クリックした瞬間の現在時刻 (HH:MM) を自動入力
- 反映:
  1. `chrome.storage.local` にローカル保存 (バックアップ兼ねる)
  2. 自己申告時間ページを開いている場合は、**今日の行の入力欄に DOM 操作で値を流し込む**
- ツールバーアイコン:
  - クリックでポップアップ → 本日の履歴 / 過去履歴 / 「現在のページに反映」ボタン
  - バッジに本日の打刻回数を表示

## インストール (開発者モード)

1. Chrome で `chrome://extensions/` を開く
2. 右上の **デベロッパーモード** をオン
3. **「パッケージ化されていない拡張機能を読み込む」** をクリック
4. このディレクトリ (`users/kou/raku-ro-puncher/`) を選択

ラクローのタブを開き直す (または再読み込み) すると、画面右下に「🕒 打刻」ボタンが出ます。

## 使い方

### 出勤

1. 朝、ラクロー (どのページでも OK) を開く
2. 右下の「🕒 打刻」ボタンをクリック
3. モーダルで「出勤」「現在時刻」を確認 → 「出勤として打刻」
4. 自己申告時間入力ページにいる場合は、その場で入力欄に時刻が反映される
   - いない場合は記録のみ。後で自己申告時間ページを開いてポップアップから「現在のページの自己申告時間に反映」を押すと一括反映できる

### 退勤 (1日中で2回目以降)

1. 退勤時に同じ「🕒 打刻」ボタンをクリック
2. 自動的に「退勤」と判定される
3. もし退勤後に再度押した場合も「退勤」として最後の打刻時刻で上書きされる

### 自己申告時間ページへの反映

- ラクローの自己申告時間入力画面を開く
- 拡張アイコンをクリック → 「現在のページの自己申告時間に反映」
- 今日の行の **1番目の入力欄に出勤時刻** / **2番目の入力欄に退勤時刻** が入る

## 自動入力のヒューリスティック

ラクローの DOM 構造が公開されていないため、以下の優先順位で「今日の行」を検出する:

1. `tr` / `[role="row"]` / `[class*="row"]` / `[class*="day"]` / `[data-date]` の中で
2. 本文に `28日` / `4/28` / `04-28` のような今日の日付表記を含むもの
3. その内部の `input[type="time"]` または `input[name*="time"]` 等

検出した行の input 配列のうち、ドキュメント順で 1 番目 = 開始 (出勤)、2 番目 = 終了 (退勤) として扱う。
React/Vue 製の場合でも値が反映されるよう、native setter 経由で `input` / `change` イベントを発火している。

DOM 構造変更で動かなくなった場合は `content.js` の `tryFillForm` を調整すればよい。

## データ保存場所

- すべて `chrome.storage.local` (拡張機能ごとにブラウザ内で隔離されたストレージ)
- 外部サーバには一切送信しない
- キー: `raku_ro_punches` = `{ "YYYY-MM-DD": [{type, time, recordedAt}, ...] }`

## 試行運用中の注意

- 来月本格運用とのことなので、まずは記録の二重保存 (拡張機能のローカル + ラクロー本体) を推奨
- DOM 自動入力が機能しない日があったら、ポップアップから手動で「反映」ボタンを押すか、ローカルストレージの履歴を見ながら手入力でフォローできます
